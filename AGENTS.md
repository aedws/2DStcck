# AGENTS.md

## Cursor Cloud specific instructions

`2dstock` is a Next.js 15 (React 19, TypeScript) client-side stock-trading simulation game. The market engine runs entirely in the browser (`src/lib/market/*`) — no backend is required to run or test the core game. Supabase is only an optional account/leaderboard layer.

### Services

Only one service is required to run the product end to end:

- **Next.js dev server** — `npm run dev` → http://localhost:3000. This serves the full game (trading, seasons, events, charts).

Standard commands are in `package.json` scripts (`dev`, `build`, `lint`, `test:cashflows`, `test:gameplay`). See `README.md` for product/feature details.

### Non-obvious notes

- **No `.env.local` needed to boot.** `src/lib/supabase/client.ts` ships hardcoded public fallback Supabase credentials, so the app runs standalone. Login, cloud wallet sync, and leaderboards gracefully degrade to LocalStorage when not logged in. Only create `.env.local` (see `.env.local.example`) if you want to point login/rankings at your own Supabase project.
- **Supabase and all "server market" edge functions are NOT required.** The `tick-market`/`trade`/`limit_orders`/cron-tick migrations and functions are legacy/deprecated (see `README.md`) — do not run them.
- The app uses Next.js static export (`output: "export"` in `next.config.ts`); production deploys as a static site to GitHub Pages. For development always use `npm run dev`, not `npm run build`/`start`.
- `src/data/generated.ts` is generated by `npm run import:companies` from `data/companies.csv` — do not edit it by hand.
